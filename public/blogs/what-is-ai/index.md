# 什么是 AI

这几年大家经常听到 AI、模型、Agent、Skill、工具调用这些词。它们听起来像一堆新概念，其实可以放在同一条线上理解：AI 是能力，模型是大脑，工具是手脚，Skill 是做事方法，Agent 是能持续完成任务的执行者。

## AI 是什么

AI 是 Artificial Intelligence，也就是人工智能。它不是某一个软件，也不是某一个网站，而是一类让机器表现出“智能行为”的技术。

传统软件通常是人把规则写死：如果用户点这个按钮，就执行这段代码；如果金额大于某个数，就触发某个流程。AI 的不同之处在于，它不是只靠固定规则工作，而是从大量数据和反馈里学到规律，然后根据当前输入做判断、生成内容或采取行动。

比如：

- 你给它一句话，它能理解大概意思。
- 你给它一张图，它能描述图里有什么。
- 你让它写代码，它能根据上下文补全实现。
- 你让它总结一篇文章，它能抓住重点。

所以更准确地说，AI 是一种“把模糊输入变成有用输出”的能力。

## 模型是什么

模型可以理解为 AI 的“大脑”。它不是人类大脑，而是一组经过训练的参数和算法。

大语言模型，比如 GPT、Claude、Gemini，核心能力是处理文本：理解、推理、生成、翻译、总结、写代码。图像模型擅长生成或理解图片。语音模型擅长听和说。不同模型就像不同专业的人，有的擅长写作，有的擅长数学，有的擅长编程，有的擅长看图。

模型本身通常只会“思考和生成”。它知道很多模式，但它不能天然访问你的电脑文件、不能自动打开网页、不能直接部署项目。要让模型真正做事，就需要工具和权限。

## Prompt 是什么

Prompt 就是你给 AI 的指令。可以是一句话，也可以是一大段背景、规则、目标和格式要求。

一个简单 prompt 可能是：

> 帮我写一段博客简介。

一个更好的 prompt 会包含更多上下文：

> 我在搭建个人技术博客，读者是刚入门的开发者。请用轻松但准确的语气，写一段 100 字以内的博客简介，避免夸张营销。

AI 的输出质量很大程度取决于 prompt。不是因为 AI “只听话”，而是因为它需要知道目标、边界、语气和上下文。

## 工具调用是什么

工具调用可以理解为给 AI 装上手脚。

模型自己不能真正读取你的本地文件，但如果系统给它一个“读取文件”的工具，它就可以请求调用这个工具。模型自己不能部署网站，但如果系统给它命令行、Vercel、Cloudflare、GitHub 等工具，它就可以在授权范围内执行部署。

在你这个博客搭建过程里，AI 做的事情就不只是聊天，而是调用了很多工具：

- 读取项目文件，理解博客模板结构。
- 修改配置和代码。
- 运行构建命令检查有没有报错。
- 推送到 GitHub。
- 部署到 Vercel 和 Cloudflare。
- 验证线上页面是否正常。

如果没有工具调用，AI 只能告诉你“应该怎么做”。有了工具调用，AI 才能真的帮你把事情做完。

## Agent 是什么

Agent 可以翻译成“智能体”或“代理执行者”。它比普通聊天机器人更进一步：它不只是回答问题，而是能围绕一个目标持续规划、执行、观察结果、修正下一步。

一个普通聊天机器人可能这样工作：

1. 你问一个问题。
2. 它回答。
3. 对话结束。

一个 Agent 更像这样工作：

1. 你给它一个目标，比如“帮我搭建博客”。
2. 它拆成步骤：准备仓库、部署博客、配置图床、测试发布。
3. 它调用工具执行。
4. 它看执行结果，如果失败就排查。
5. 它继续下一步，直到目标完成。

所以 Agent 的重点不是“更会聊天”，而是“更会完成任务”。

## Skill 是什么

Skill 可以理解为 Agent 的专项技能说明书。

如果说模型是大脑，工具是手脚，那么 Skill 就像“工作手册”。它告诉 Agent 遇到某类任务时应该按什么流程做、注意什么风险、使用哪些工具、怎样验证结果。

比如：

- 部署 Cloudflare 时，要先确认登录状态、资源绑定、构建目录。
- 做前端页面时，要注意移动端、按钮尺寸、文字溢出。
- 调试 bug 时，要先找根因，不要乱试。
- 操作 GitHub 时，要避免提交密钥。

Skill 的价值是让 Agent 不只是“聪明”，还更稳定、更像一个有经验的人在按流程做事。

## AI、Agent、Skill 的关系

可以用一个简单比喻：

- AI：一种智能能力。
- 模型：提供智能能力的大脑。
- Prompt：你给大脑的任务说明。
- 工具：让它能操作外部世界的手脚。
- Skill：告诉它某类任务怎么做的工作手册。
- Agent：把大脑、工具、记忆、步骤和 Skill 组合起来，持续完成目标的执行者。

这几个词不是互相替代，而是层层组合。

## 为什么现在 Agent 很重要

因为很多真实任务不是一句话能完成的。

“解释一个概念”只需要模型回答就够了；但“搭建一个博客”就需要很多步骤：创建仓库、配置环境变量、部署、排查报错、验证链接、保存密钥、写文章。这类任务需要连续动作和反馈修正，Agent 才真正有优势。

Agent 的核心价值是把 AI 从“回答者”变成“协作者”。它不是只给建议，而是能和你一起把事情推进。

## 需要注意什么

AI 很有用，但它不是魔法。

第一，它可能会犯错。尤其是在信息过期、上下文不完整、工具结果不明确的时候。

第二，权限很重要。能操作文件、仓库、云平台的 AI，一定要有边界。比如密钥不能提交到公开仓库，删除文件前要确认范围。

第三，验证很重要。AI 说“完成了”不够，必须实际运行构建、打开页面、检查返回结果。

第四，人仍然是决策者。AI 可以帮你执行、解释、整理，但网站要叫什么、文章要保留什么、哪些信息能公开，最终都应该由你决定。

## 总结

AI 是能力，模型是大脑，工具是手脚，Skill 是方法，Agent 是能持续做事的执行者。

如果只是聊天，AI 像一个知识助手；如果能调用工具并按流程完成目标，它就更像一个数字协作者。未来很多软件不会只是给你一堆按钮，而是让你说出目标，然后由 Agent 帮你拆解、执行、验证。

真正重要的不是这些名词有多新，而是它们让普通人也能更低成本地完成复杂事情。比如搭建一个博客、部署一个图床、发布一篇文章。这些以前需要你一步步学，现在可以有人陪你一起做。
